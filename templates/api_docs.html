Introduction

The Job API provides a RESTful interface for working with job listings. It supports standard CRUD operations (Create, Read, Update, Delete) as well as search functionality.

Base URL

https://your-domain.replit.app/api

Response Format

All responses are returned in JSON format. Successful requests will return either a single job object or an array of job objects.

Endpoints

GET

/jobs

Retrieves a list of all job listings.

Example Request:

GET /api/jobs

Example Response:

[
  {
    "id": 1,
    "title": "Software Engineer",
    "company": "Tech Solutions",
    "location": "San Francisco, CA",
    "salary": "$120,000 - $150,000",
    "url": "https://example.com/jobs/1",
    "created_at": "2025-03-15T10:30:00"
  },
  {
    "id": 2,
    "title": "Product Manager",
    "company": "Innovative Inc.",
    "location": "New York, NY",
    "salary": "$110,000 - $140,000",
    "url": "https://example.com/jobs/2",
    "created_at": "2025-03-14T09:15:00"
  }
]
GET

/jobs/{id}

Retrieves a specific job listing by its ID.

Parameters:

  • id (path parameter, required) - The ID of the job listing

Example Request:

GET /api/jobs/1

Example Response:

{
  "id": 1,
  "title": "Software Engineer",
  "company": "Tech Solutions",
  "location": "San Francisco, CA",
  "salary": "$120,000 - $150,000",
  "url": "https://example.com/jobs/1",
  "created_at": "2025-03-15T10:30:00"
}
POST

/jobs

Creates a new job listing.

Request Body:

  • title (string, required) - Job title
  • company (string, required) - Company name
  • location (string, required) - Job location
  • salary (string, optional) - Salary range or description
  • url (string, optional) - URL to the job listing

Example Request:

POST /api/jobs
Content-Type: application/json

{
  "title": "Backend Developer",
  "company": "Cloud Services",
  "location": "Austin, TX",
  "salary": "$100,000 - $130,000",
  "url": "https://example.com/jobs/backend-dev"
}

Example Response:

{
  "id": 3,
  "title": "Backend Developer",
  "company": "Cloud Services",
  "location": "Austin, TX",
  "salary": "$100,000 - $130,000",
  "url": "https://example.com/jobs/backend-dev",
  "created_at": "2025-03-15T15:45:00"
}
PUT

/jobs/{id}

Updates an existing job listing.

Parameters:

  • id (path parameter, required) - The ID of the job listing to update

Request Body:

Include any fields you want to update:

  • title (string, optional) - Job title
  • company (string, optional) - Company name
  • location (string, optional) - Job location
  • salary (string, optional) - Salary range or description
  • url (string, optional) - URL to the job listing

Example Request:

PUT /api/jobs/3
Content-Type: application/json

{
  "salary": "$110,000 - $140,000",
  "location": "Remote"
}

Example Response:

{
  "id": 3,
  "title": "Backend Developer",
  "company": "Cloud Services",
  "location": "Remote",
  "salary": "$110,000 - $140,000",
  "url": "https://example.com/jobs/backend-dev",
  "created_at": "2025-03-15T15:45:00"
}
DELETE

/jobs/{id}

Deletes a job listing.

Parameters:

  • id (path parameter, required) - The ID of the job listing to delete

Example Request:

DELETE /api/jobs/3

Example Response:

{
  "message": "Job 3 deleted successfully"
}

POST

/jobs/bulk

Creates multiple job listings in a single request.

Request Body:

An array of job objects, each containing:

  • title (string, required) - Job title
  • company (string, required) - Company name
  • location (string, required) - Job location
  • salary (string, optional) - Salary range or description
  • url (string, optional) - URL to the job listing

Example Request:

POST /api/jobs/bulk
Content-Type: application/json

[
  {
    "title": "Data Scientist",
    "company": "Analytics Co",
    "location": "Chicago, IL",
    "salary": "$130,000 - $160,000",
    "url": "https://example.com/jobs/data-scientist"
  },
  {
    "title": "UX Designer",
    "company": "Creative Design",
    "location": "Portland, OR",
    "salary": "$90,000 - $115,000",
    "url": "https://example.com/jobs/ux-designer"
  }
]
